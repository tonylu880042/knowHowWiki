## 📋 精華摘要（Key Takeaways）

### 1. 整套方法論的本質是「把 AI 當成 Junior 工程師來帶」

不是學新工具、不是換新範式。你花 20 年學的軟體工程原則——架構設計、回饋迴圈、模組化、可測試性——全部照搬，只是執行者從「你的手指」變成了「AI 代理」。人類退居到兩個 AI 做不到的位置：**定義需求** 和 **驗收品質**。

### 2. 五個階段形成一個不斷收窄的「漏斗」

```
模糊想法（寬） → Grill Me 精煉 → Ubiquitous Language 校準 
→ PRD 文件化 → Issues 拆解 → Ralph Loop 實作（窄）
```

每一步都是在**減少歧義**。到 Ralph Loop 接手時，任務已經被壓縮成了非常具體的垂直切片，AI 幾乎不可能跑偏。

### 3. 「Collocation」是 LLM 工作流的隱藏殺手鐗

Grill Me 之所以有效，不只是因為「問了好問題」，更因為問題和答案在 context 中**緊密相鄰**。Transformer 的注意力機制對距離敏感——相鄰的資訊會被更強烈地關聯。這解釋了為什麼不用 `ask_user_question` 工具（會插入 JSON 包裝，拉開問與答的距離）。

### 4. Ubiquitous Language 在 AI 時代的價值被放大了十倍

在人與人溝通中，Ubiquitous Language 減少誤解。但在人與 AI 溝通中，它還額外解鎖了：
- AI 搜尋碼庫時使用正確的關鍵字
- 跨 session 的語義一致性（不同的 Claude 實例都能理解）
- 函式命名的統一性（`materializeGhost()` 而非各種別名）

### 5. Spec-to-Code 的根本問題：邊界案例只會在 QA 中浮現

QA 階段的存在不是因為 AI 寫的程式碼差，而是因為**有些需求在你親手操作之前根本不會被想到**。例如「如果目標路徑不是 Git repo 會怎樣？」——這個 edge case 在 Grill Me 的 22 分鐘盤問中完全沒被觸及，只有在實際操作 UI 時才浮現。

## 🎙️ 語音導覽草稿（Podcast Script）

*以下為口語化講稿，適合丟給語音模型朗讀。*

---

嘿，歡迎回來。今天我要跟你聊一套我最近學到的超酷的 AI 開發方法論。

你知道很多人在說 AI 寫程式是全新的典範轉移嗎？這套方法論的核心觀點是——「不，才不是」。過去 20 年我們學的軟體工程最佳實踐，用在 AI 工具上反而效果更好。

讓我用五個步驟來畫這個大圖：

**第一步叫 Grill Me。** 你對 AI 口述一個模糊的想法，然後 AI 反過來盤問你，像一個嚴格的技術主管那樣。它會問你邊界案例、問你權衡、問你為什麼要做這個功能。影片裡有一個很棒的例子——22 分鐘的盤問，最後收斂出 8 個清晰的需求點。

這裡有一個很微妙但很重要的技巧叫 Collocation——問題和答案緊貼在一起，讓 Transformer 的注意力機制能同時看到兩者。這就是為什麼不用專門的「問答工具」，直接在聊天裡來回比較好。

**第二步是建立 Ubiquitous Language。** 這來自 Domain-Driven Design 那本書。你維護一份專案術語表——比如「Materialize」代表把幽靈實體寫入磁碟，「Materialization Cascade」代表在幽靈課程裡觸發的連鎖實體化動作。有了這個，以後你可以直接說「實體化串聯有 bug」，AI 秒懂。

**第三步是 PRD 到 Issues。** AI 把精煉後的需求寫成產品規格書，然後拆成 GitHub Issues。這裡有一個很有趣的信任邊界——PRD 可以不看直接接受，因為 AI 很擅長摘要。但 Issue 的粒度必須自己調，因為太小的任務不值得啟動一個代理，太大的任務會讓代理失焦。

**第四步是 Ralph Loop。** 你按下 `pnpm ralph`，然後就可以去散步了。真的。這個 Docker 容器裡的 AI 代理會自動撿 GitHub Issue、寫程式碼、跑測試、commit、關 Issue，然後撿下一個。影片裡 5 輪迭代產出了 6 個 commit，整個功能花了大約一個半小時——但人類在這段時間裡去散步、喝茶、跟父母聊天了。

**最後一步是 QA 迴圈。** 你回來之後手動測試，用 App 裡的反饋按鈕直接提交 bug，然後又啟動一輪 Ralph Loop 讓 AI 在背景修復，同時你繼續測試其他功能。8 分鐘內提交了 6 個 Issue。

整個模式被稱為「日班 / 夜班」——人類做日班，負責構想和審查；AI 做夜班，負責實作和修復。兩班同時運轉，互不阻塞。

最精華的洞察是這句話：「一旦我們完成了想法的思考，我們的工作基本上就結束了——直到需要 QA 輸出為止。」

這代表什麼？人類的稀缺資源不是打字速度，是決策力。而這套方法論把人類的時間集中在了決策上，把一切可以委派的都交給了 AI。

好的，今天就到這裡。下次再聊！
